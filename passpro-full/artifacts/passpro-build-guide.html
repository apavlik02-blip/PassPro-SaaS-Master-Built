Build a SaaS Platform
with AI Tools — Zero to Launch

A complete, beginner-friendly guide to building PassPro — a Wisconsin Life Insurance exam prep SaaS — from scratch using AI tools like Claude, Cursor, and v0. No prior SaaS experience required. Every step tells you exactly what to do, what to type, and which AI tool to use.

What You're Building

PassPro is a fully AI-powered exam prep platform for Wisconsin Life Insurance licensing. By the end of this guide you'll have a live, paying SaaS with: a student dashboard, AI study companion, spaced repetition flashcards, a full 100-question exam simulator, agency admin tools, Stripe payments, and OCI compliance tracking.

01
Phase 1 · Weeks 1–2

Project Setup &
Architecture

Install your tools, create your project, and set up the folder structure. By the end of Phase 1 you'll have a working Next.js app running on your computer.

⏱ 1–2 weeks 🤖 Claude + Cursor 🎯 Goal: Local app running
1
Install Node.js and create your Next.js project
Node.js is the engine that runs your app. Download it from nodejs.org (click the LTS version). Then open Terminal (Mac) or Command Prompt (Windows) and run this command to create your PassPro project.
npx create-next-app@latest passpro --typescript --tailwind --eslint --app --src-dir cd passpro
💡 When it asks questions during setup, press Enter to accept all the defaults. The --app flag uses the modern Next.js App Router which this guide is built around.
2
Install Cursor and open your project
Go to cursor.sh and download Cursor. It looks just like VS Code. Open your passpro folder in Cursor. From now on, all coding happens here — you'll describe what you want and Cursor writes the code.
🤖Your first Cursor prompt
"I'm building a Wisconsin Life Insurance exam prep SaaS called PassPro. It uses Next.js 14 App Router, TypeScript, Tailwind CSS, Supabase for the database, Clerk for auth, Stripe for payments, and the Anthropic Claude API for AI features. Please show me the recommended folder structure and explain what each folder does."
3
Install all dependencies
These are the packages your app needs to run. Copy and paste this command into your Terminal inside the passpro folder.
npm install @supabase/supabase-js @supabase/ssr @clerk/nextjs stripe @stripe/stripe-js @anthropic-ai/sdk lucide-react clsx tailwind-merge
💡 If you see warnings in red, that's usually fine. Only errors (the build fails) need fixing. Ask Claude: "I got this error installing npm packages: [paste the error]"
4
Set up your GitHub repository
GitHub is where your code lives safely online. Go to github.com, create a free account, then create a new repository called passpro. Then connect your local project to GitHub.
git init git add . git commit -m "Initial PassPro setup" git remote add origin https://github.com/YOURUSERNAME/passpro.git git push -u origin main
💡 Replace YOURUSERNAME with your actual GitHub username. After this, every time you finish a chunk of work, run git add . && git commit -m "description" && git push to save it.
5
Create your environment variables file
Environment variables are secret keys that connect your app to services like Supabase and Stripe. Create a file called .env.local in your project root. You'll fill in the values as you set up each service.
# Supabase NEXT_PUBLIC_SUPABASE_URL=your_supabase_url_here NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here # Clerk Authentication NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_key_here CLERK_SECRET_KEY=your_clerk_secret_here # Stripe Payments NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=your_stripe_pub_key_here STRIPE_SECRET_KEY=your_stripe_secret_here STRIPE_WEBHOOK_SECRET=your_webhook_secret_here # Anthropic Claude API ANTHROPIC_API_KEY=your_anthropic_key_here
⚠️ Never share this file. Never commit it to GitHub. It's already in your .gitignore file so this is automatic — but double-check.
6
Run the app and confirm it works
Start your development server and open the app in your browser. You should see the default Next.js welcome page.
npm run dev
💡 Open your browser and go to http://localhost:3000. If you see a Next.js welcome page, Phase 1 Step 6 is done! Leave this running while you work — it auto-refreshes whenever you save a file.

Phase 2 · Weeks 2–3

Database Setup
with Supabase

Create your database, design the tables that store student data, questions, and compliance hours. Supabase gives you a full PostgreSQL database with a visual interface — no SQL experience required.

⏱ 1 week 🤖 Claude + Supabase 🎯 Goal: Database live with all tables
7
Create your Supabase project
Go to supabase.com and create a free account. Click "New Project", name it passpro, choose a region close to you, and create a strong database password (save it somewhere). Wait ~2 minutes for it to set up.
💡 After setup, go to Project Settings → API. Copy your Project URL and anon/public key into your .env.local file where it says your_supabase_url_here etc.
8
Create the database tables with Claude
Ask Claude to write the SQL for all your PassPro tables. Then go to Supabase Dashboard → SQL Editor and run it.
🤖Claude prompt — Database Schema
"Write the complete Supabase PostgreSQL SQL to create all tables for PassPro, a Wisconsin Life Insurance exam prep SaaS. I need tables for: users (with plan, state, exam_date, readiness_pct, hours_logged, streak_days, xp), units (curriculum units 1-8), questions (exam questions with domain, topic, options array, correct_answer, explanation, is_wi_specific), user_answers (tracks student answers), flashcards (front, back, unit, topic, is_wi_specific), flashcard_progress (SM-2 spaced repetition: interval, ease_factor, reps, next_review per user per card), exam_attempts (score, pct, time_used, domain_scores JSON), compliance_sessions (user_id, duration_minutes, unit_id, date), and agencies (name, plan, seat_count). Include Row Level Security policies so users can only see their own data."
💡 Copy Claude's SQL output, go to Supabase → SQL Editor, paste it, and click Run. If you get errors, paste them back to Claude and ask it to fix them.
9
Seed the database with Wisconsin curriculum content
Ask Claude to generate seed data — the 8 curriculum units and a starter set of exam questions — and insert it into your database.
🤖Claude prompt — Seed Data
"Write SQL INSERT statements to seed my PassPro database with: (1) All 8 Wisconsin Life Insurance curriculum units with titles, hours, and lesson counts. (2) 20 sample exam questions across 4 domains (Insurance Regulation, Life Insurance Basics, Annuities, Policy Provisions) in the format my questions table expects. Include 5 Wisconsin-specific questions marked with is_wi_specific=true. Make them realistic PSI Series 22-01 exam questions."
10
Connect your Next.js app to Supabase
Ask Cursor to create the Supabase client files so your app can talk to the database.
🤖Cursor prompt
"Create the Supabase client setup for my Next.js 14 App Router project. I need: (1) src/lib/supabase/client.ts for browser-side queries, (2) src/lib/supabase/server.ts for server-side queries using the SSR package, (3) src/middleware.ts to refresh auth sessions. Use the @supabase/ssr package."

Build the student dashboard

It should: (1) Fetch the logged-in user's data from Supabase (readiness_pct, streak_days, xp, hours_logged, exam_date, weak_topics array), (2) Display a hero section with personalized greeting using their name, (3) Show 4 stat cards: Readiness %, Streak days, XP total, Hours logged, (4) Show curriculum progress bars for all 8 units fetched from the units table, (5) Show an exam countdown, (6) Show weak areas as clickable chips. Build the study modules page
Lists all 8 curriculum units. Students click a unit to see lessons. Progress is tracked and stored in Supabase.

Mark completed lessons with a checkmark. Track which lesson the user is on and store their progress in Supabase when they complete a lesson. Use Server Components for data fetching where possible."
15
Build the practice exam system
The core learning engine. Students answer questions, get explanations, and their scores are stored. Supports 4 modes: Knowledge Check (3Q), Module Quiz (5Q), Unit Test (10Q), and Exam Simulation (100Q timed).
🤖Cursor prompt
"Build the PassPro practice exam system at src/app/practice/. Create: (1) src/app/practice/page.tsx — mode selector showing 4 cards (Knowledge Check, Module Quiz, Unit Test, Exam Simulation), (2) src/app/practice/[mode]/page.tsx — the actual quiz interface that fetches questions from Supabase filtered by domain, displays one question at a time with 4 answer options, shows domain/topic badges, allows flagging questions, tracks answers in React state, has a progress bar, and on completion shows results with domain breakdown. (3) An API route at src/app/api/practice/submit/route.ts that saves the exam attempt to Supabase including domain_scores JSON. The exam simulation mode gets a 2-hour countdown timer."
16
Build the flashcard system with SM-2 spaced repetition
Students flip cards, rate themselves, and the SM-2 algorithm schedules the next review. All progress stored in Supabase.
🤖Cursor prompt
"Build the flashcard system at src/app/flashcards/. Include: (1) A deck selection page showing cards Due Today, New Cards, WI Only, and All Cards — counts fetched from Supabase, (2) The study session page that shows one card at a time (front = question, click to flip to answer), (3) After flipping, show 4 rating buttons: Again (0), Hard (1), Good (2), Easy (3), (4) An API route at src/app/api/flashcards/rate/route.ts that implements the SM-2 algorithm — updating interval, ease_factor, reps, and next_review in the flashcard_progress table, (5) XP is awarded per card rated. The SM-2 formula: if quality < 2, reset to interval=1, reps=0. Otherwise interval = prev_interval * ease_factor, ef = max(1.3, ef + 0.1 - (3-q)*(0.08+(3-q)*0.02))."
17
Build the OCI compliance tracker
Wisconsin requires 20 verified hours before sitting the exam. Every study session is logged with a timestamp. Students can see their exact progress.
🤖Cursor prompt
"Build the OCI compliance tracker at src/app/compliance/. It should: (1) Show two progress bars — total seat time (8.5/20h) and WI Laws & Ethics sub-requirement (1/8h), (2) Fetch all compliance_sessions for the user from Supabase and show a session log table (date, unit studied, minutes, type), (3) Lock the Exam Simulation feature if hours < 20 — show a lock icon and message, (4) Create a server action that logs a new session whenever a student completes a study session or quiz, with inactivity detection (pause timer if no interaction for 30 minutes), (5) Show a certificate eligibility status — green checkmark when 20h is reached."
🏛 Wisconsin OCI Note: The 20-hour requirement is mandated by Wis. Stat. § 628.03. Your compliance tracker must record timestamps per session — this is what OCI providers submit to the state. Each session must have start time, end time, and unit studied.

Get your Anthropic API key

Create a streaming API endpoint that powers the AI Coach chat. Streaming means words appear one at a time — much better experience than waiting for the whole response.

Phase 6 · Weeks 10–12

Deploy &
Go Live

Deploy PassPro to the internet, connect your domain, set up monitoring, and launch to your first users. This is the phase where it goes from a local project to a real product.

⏱ 2 weeks 🤖 Claude + Vercel 🎯 Goal: First paying customer
27
Deploy to Vercel
Go to vercel.com, create a free account, and connect your GitHub repository. Vercel automatically deploys your app whenever you push code to GitHub. It takes about 2 minutes.
💡 In Vercel: New Project → Import your GitHub repo → it detects Next.js automatically. Before clicking Deploy, add all your environment variables from .env.local in the "Environment Variables" section. Never deploy without these or your app won't connect to any services.
28
Connect your domain
Buy a domain (e.g. passprowisconsin.com) from Namecheap or Google Domains (~$12/year). Then connect it to Vercel so your app is live at your custom domain.
💡 In Vercel: Settings → Domains → Add Domain → enter your domain name. Vercel gives you DNS records to add in your registrar. In Namecheap: Domain List → Advanced DNS → add the records Vercel shows you. Takes 10–60 minutes to go live.
29
Update all webhook URLs to production
Your Stripe and Clerk webhooks are still pointing to localhost. Update them to your live domain.
# Update these in each dashboard: Stripe webhook: https://yourdomain.com/api/webhooks/stripe Clerk webhook: https://yourdomain.com/api/webhooks/clerk
⚠️ This is a common launch mistake. If you forget to update webhooks, payments will process but your database won't update — users will pay and not get access. Test a live payment immediately after launch.
30
Switch Stripe from test mode to live mode
In Stripe, toggle the switch from "Test mode" to "Live mode". Regenerate your API keys in live mode and update them in Vercel's environment variables. Recreate your products in live mode.
💡 In Vercel: Project Settings → Environment Variables → update STRIPE_SECRET_KEY and NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY with your live mode keys. Click Redeploy after updating.
31
Set up error monitoring with Sentry
Sentry tells you when something breaks in production. Go to sentry.io, create a free account, and add it to your app. You'll get an email whenever a user hits an error.
🤖Cursor prompt
"Add Sentry error monitoring to my Next.js 14 PassPro app. Install @sentry/nextjs, run the Sentry wizard (npx @sentry/wizard@latest -i nextjs), and configure it to capture unhandled errors in both client and server components. Add my Sentry DSN to environment variables."
